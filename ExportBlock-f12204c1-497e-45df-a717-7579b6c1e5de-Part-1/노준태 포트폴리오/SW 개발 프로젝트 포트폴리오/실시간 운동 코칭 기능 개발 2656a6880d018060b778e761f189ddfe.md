- 개발 과정
    
    
    ## **1. 문제 인식**
    
    - **사용자가 운동할 때 위치가 바뀌거나 체형이 달라도 정확한 운동 분석이 어려운 문제가 발생.**
    - 예: 사용자가 카메라 앞에서 좌우로 움직이면 목표 포즈와 비교가 어려워지고, 어깨가 넓은 사람과 좁은 사람이 같은 운동을 해도 분석 기준이 달라짐.
    
    ---
    
    ## **2. 초기 접근 (캘리브레이션)**
    
    - **사용자가 측정 시작 전에 기준 포즈(캘리브레이션)를 취하도록 설계.**
    - 이때 사용자의 체형 비율(어깨 너비, 다리 길이 등)을 계산하여 개인화된 분석 기준을 설정.
    - 이를 통해 "사용자 고유 비율"을 운동 분석에 반영 → 기본적인 체형 차이 문제 완화.
    
    **구현된 기능:**
    
    - segment_calibrate_user(): 사용자 기본 포즈로 개인화 캘리브레이션
    - 어깨 너비 기준 스케일 팩터 자동 계산
    - 중심점 오프셋 계산으로 위치 정규화
    - 캘리브레이션 품질 평가 시스템
    
    ---
    
    ## **3. 중간 시도 (비율 매핑)**
    
    - 단순 캘리브레이션만으로는 사용자가 실시간으로 움직일 때 목표 포즈가 고정되어 있어서 정확한 비교가 어려움
    - **상반신/하반신 구간별 비율 매핑**을 시도.
    - 예: 상체는 어깨–팔–손목 비율 기준, 하체는 골반–무릎–발목 비율 기준.
    - 하지만 동작이 복잡해질수록 오차 누적 문제가 생김.
    
    **구현된 기능:**
    
    - segment_analyze_smart(): 실시간 사용자 위치 기준 목표 포즈 생성
    - 현재 포즈의 어깨 너비를 실시간으로 측정하여 목표 포즈에 적용
    - 발 중심점 기준으로 포즈 위치 자동 조정
    
    ---
    
    ## **4. 최종 해결책 (스마트 포즈 분석)**
    
    - 최종적으로 **사용자가 움직여도 알아서 따라오는 스마트 포즈 분석**을 구현.
    - 목표 포즈를 사용자의 현재 위치와 체형에 맞게 실시간으로 조정하여 정확한 비교 분석.
    - 이로써 사용자가 어디에 서있든, 체형이 어떻든 **정확한 운동 진행도 판별**이 가능.
    
    **구현된 기능:**
    
    - segment_analyze_smart(): 사용자 위치에 맞춰 목표 포즈 자동 조정
    - 실시간 어깨 너비 측정으로 크기 자동 조절
    - 발 중심점 기준으로 위치 자동 맞춤
    - calculate_segment_progress(): 조정된 목표 포즈로 정확한 진행도 계산
    - segment_calculate_similarity(): 실시간 유사도 분석
    
    ---
    
    ## **5. 결과**
    
    - **사용자가 움직여도 알아서 따라오는 스마트 포즈 분석 가능**
    - **체형에 관계없이 정확한 운동 진행도 판별**
    - **실시간 크기 조절 및 위치 맞춤**
    
    **추가로 해결한 문제들:**
    
    ### **6. 사용자 역할 분리 (v2.0.0)**
    
    - **A 이용자(기록자)와 B 이용자(사용자) 역할을 명확히 분리**
    - A는 이상적 비율로 포즈를 기록하고 JSON 파일로 저장
    - B는 개인 체형에 맞게 변환된 포즈로 실시간 분석
    
    ### **7. JSON 기반 워크아웃 관리**
    
    - **포즈 데이터를 JSON 파일로 체계적으로 관리**
    - segment_load_all_segments(): 모든 포즈를 미리 로드하여 성능 최적화
    - segment_set_current_segment(): 인덱스 기반으로 세그먼트 빠른 선택
    
    ### **8. 실시간 성능 최적화**
    
    - **60fps 목표 달성 가능한 실시간 처리**
    - 메모리 사용량 < 10MB
    - 진행도 정확도: ±5% 오차 범위
    
    ### **9. 크로스 플랫폼 지원**
    
    - **iOS, Android, Windows, macOS, Linux 모두 지원**
    - CMake 기반 빌드 시스템
    - Swift 브리징 헤더로 iOS 통합 최적화
    
    ### **10. 포괄적인 테스트 시스템**
    
    - **단위 테스트, 통합 테스트, 성능 검증 완료**
    - 모든 예외 상황 처리
    - 메모리 누수 방지

    캘리브레이션
    
    ```csharp
    // 어깨 너비 기준 스케일 팩터 계산
    float user_shoulder_width = distance_3d(
        &base_pose->landmarks[POSE_LANDMARK_LEFT_SHOULDER].position,
        &base_pose->landmarks[POSE_LANDMARK_RIGHT_SHOULDER].position
    );
    
    // 이상적 어깨 너비 (실제 데이터 기반: ~322.78)
    float ideal_shoulder_width = 322.78f;
    
    // 스케일 팩터 계산
    g_user_calibration.scale_factor = user_shoulder_width / ideal_shoulder_width;
    ```
    
    사용자 위치에 맞는 타겟 포즈
    
    ```csharp
    int segment_analyze_smart(const PoseData *current_pose, float *out_progress,
                              float *out_similarity, bool *out_is_complete,
                              Point3D *out_corrections,
                              PoseData *out_smart_target_pose) {
      
      // 1. 현재 포즈의 크기 측정 (어깨 너비 기준)
      float current_shoulder_width = 0;
      if (current_left_shoulder.inFrameLikelihood >= 0.5f &&
          current_right_shoulder.inFrameLikelihood >= 0.5f) {
        float dx = current_left_shoulder.position.x - current_right_shoulder.position.x;
        float dy = current_left_shoulder.position.y - current_right_shoulder.position.y;
        current_shoulder_width = sqrtf(dx * dx + dy * dy);
      }
    
      // 2. 목표 포즈의 크기 측정
      float target_shoulder_width = 0;
      // ... 동일한 로직
    
      // 3. 크기 조정을 위한 스케일 계산
      float scale = (target_shoulder_width > 0)
                        ? current_shoulder_width / target_shoulder_width
                        : 1.0f;
    
      // 4. 현재 사용자의 발 중심점 계산
      Point3D current_foot_center = {0};
      if (current_left_ankle.inFrameLikelihood >= 0.3f &&
          current_right_ankle.inFrameLikelihood >= 0.3f) {
        current_foot_center.x = (current_left_ankle.position.x + current_right_ankle.position.x) / 2.0f;
        current_foot_center.y = (current_left_ankle.position.y + current_right_ankle.position.y) / 2.0f;
        current_foot_center.z = (current_left_ankle.position.z + current_right_ankle.position.z) / 2.0f;
      }
    
      // 5. 스마트 목표 포즈 조정
      // 먼저 발 중심점을 원점으로 이동 (스케일링 기준점)
      for (int i = 0; i < POSE_LANDMARK_COUNT; i++) {
        out_smart_target_pose->landmarks[i].position.x -= target_foot_center.x;
        out_smart_target_pose->landmarks[i].position.y -= target_foot_center.y;
        out_smart_target_pose->landmarks[i].position.z -= target_foot_center.z;
      }
    
      // 크기 조정 (스케일링)
      for (int i = 0; i < POSE_LANDMARK_COUNT; i++) {
        out_smart_target_pose->landmarks[i].position.x *= scale;
        out_smart_target_pose->landmarks[i].position.y *= scale;
        out_smart_target_pose->landmarks[i].position.z *= scale;
      }
    
      // 현재 사용자의 발 중심점으로 이동
      for (int i = 0; i < POSE_LANDMARK_COUNT; i++) {
        out_smart_target_pose->landmarks[i].position.x += current_foot_center.x;
        out_smart_target_pose->landmarks[i].position.y += current_foot_center.y;
        out_smart_target_pose->landmarks[i].position.z += current_foot_center.z;
      }
    
      // 6. 조정된 목표 포즈로 정확한 분석 수행
      float progress = calculate_segment_progress(current_pose, &smart_start_pose,
                                                  out_smart_target_pose, NULL, 0);
      bool completed = is_segment_completed(current_pose, out_smart_target_pose, NULL, 0, 0.8f);
      float similarity = segment_calculate_similarity(current_pose, out_smart_target_pose);
      calculate_correction_vectors(current_pose, out_smart_target_pose, NULL, 0, out_corrections);
    
      *out_progress = progress;
      *out_is_complete = completed;
      *out_similarity = similarity;
    
      return SEGMENT_OK;
    }
    ```
    

## 📌 프로젝트 개요

재직 중 진행한 프로젝트. Flutter 기반으로 사용자의 운동 자세를 실시간 분석하여 횟수를 자동으로 카운팅하는 코칭 기능 개발.
