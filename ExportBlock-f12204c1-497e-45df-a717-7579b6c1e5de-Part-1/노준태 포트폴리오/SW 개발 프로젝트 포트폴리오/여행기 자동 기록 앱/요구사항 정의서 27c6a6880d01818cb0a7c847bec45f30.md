## 2-2. 데이터 연동 규칙

---

### 1) 실시간 연동 데이터

- 위치 데이터 : Flutter → Supabase 직접 저장
- 사용자 인증 : Supabase Auth SDK
- 이미지 업로드 : Supabase Storage SDK

### 2) **배치 연동**

- 관광지 정보 : 일 1회 업데이트
- 날씨 정보 : 시간당 1회 업데이트
- 사용자 행동 분석 : 주 1회 배치 처리

### 3) **캐싱 전략**

- 관광지 정보: Redis 24시간 캐싱
- 날씨 정보: 메모리 1시간 캐싱
- API 응답: 상황별 5분-1시간 캐싱

# 3. 제약사항 및 고려사항

---

## 3-1. 기술적 제약사항

---

### **1) 위치 추적 제약**

- GPS 음영지역에서 정확도 저하
- iOS 백그라운드 위치 추적 제한
- 배터리 소모 최적화 필요

### 2) AI 처리 제약

- OpenAI API (혹은 기타 LLM API) 비용 및 호출 제한
- 네트워크 불안정 시 처리 지연
- 한국어 감정 분석 정확도 한계

## 3-2. 사용성 고려사항

---

### **1) 프라이버시 보호**

- 위치 추적 명시적 동의 절차
- 정확한 GPS 좌표 저장 금지
- 사용자 개인정보 최소 수집

### **2) 사용자 부담 최소화**

- 관광지가 아닌 곳에서의 기록 요청 최소화
- 사진 촬영 시점에만 감정 입력 제안
- 자동 처리 실패 시 수동 보정 옵션 제공

## 3-3. 법적/정책적 고려사항

---

### **1) 개인정보보호**

- 개인정보보호법 준수
- 위치정보보호법 준수
- 데이터 처리 동의 절차 명확화

### **2) 외부 API 이용약관**

- TourAPI 이용 조건 준수
- 기상청 API 이용 제한 준수
- OpenAI API 사용 정책 준수

# 4. 검증 및 테스트 요구사항

---

## 4-1. 기능 검증 기준

---

### **핵심 플로우 테스트**

- 여행 시작 → 감정 입력 → 카드 생성 → 여행기 완성 전체 플로우
- 위치 이동 감지 및 구간 분할 정확성
- AI 감정 분석 결과 품질 및 일관성
- 외부 API 연동 안정성

### **성능 테스트**

- 다양한 네트워크 환경에서의 응답시간
- 동시 사용자 부하 테스트
- 배터리 소모량 측정
- 메모리 사용량 최적화 확인

## 4-2. 사용자 검증

---

### **사용성 테스트**

- 타겟 페르소나 대상 사용성 평가
- 첫 사용자 온보딩 완료율 측정
- 감정 입력 완료율 및 만족도 조사
- 자동 생성 결과에 대한 사용자 만족도

### **A/B 테스트 계획**

- 감정 입력 방식 (텍스트 vs 음성 우선)
- AI 생성 결과 표시 방식
- 개인화 추천 알고리즘 효과
