# GEN_001

상태: 초안 작성
구분: 여행기 생성
기능 간단 설명: 여행 기록 종료 후, 사용자가 입력했던 감정 문장을 AI로 분석하여 태그 생성
기능명: 감정 분석 및 태그 생성
우선 순위: 1순위

### ✅ 여행 중 카드

*목적: 기록이 잘 되고 있다는 **즉각적 피드백 제공** → 사용자와 앱의 상호작용 유지*

| 요소 | 설명 | 예시 |
| --- | --- | --- |
| 장소명 (근사 관광지) | 사용자의 위치 근사화 | 봉포항 근처 |
| 한줄평  | 관광지 한줄평  |  |
| 해시 태그 (또는 감성 문장) | 사용자가 남긴 감정 → AI가 문장 변환 | “평온한”, “시원한” |
| 시간 | 입력된 시각 (또는 `몇 분 전`) | 오전 10:41 |
| 대표 이미지 (있으면) | 사진이 있을 경우 썸네일로 표시 | 사진 1장 |
| 표시 스타일 | 진행 중인 느낌 강조 (실시간 반응) | 상단 “기록됨” 라벨, 붉은 점 표시 등 |
| 장소 설명 (선택) | TourAPI 기반 장소 한 줄 설명 | “동해안 대표 해변, 맑은 물과 백사장” |

**추가 생각**

---

**AI 태그 기준** 

- AI가 문장을 변환할 때, 어떤 방식으로 변환할 것인지 생각해봐야 할 듯.

- 태그나 키워드를 생각해두고, AI에게 매핑되는 태그를 골라라고 할 것인지, 자유도를 부여해서 AI가 알아서 뽑아줄지 생각 

**카드 생성 시점** 

- 감정 기록 입력, 사진 업로드, 일정 체류 감지를 우선적으로 생각
- 사진을 업로드 하거나 감정 기록을 하기 위해선 사용자의 개입이 필요함 → 서비스가 알아서 카드 업로드 자동화를 생가했지만, 실제적으로는 어느정도의 사용자 개입 필요하지 않나는 생각
