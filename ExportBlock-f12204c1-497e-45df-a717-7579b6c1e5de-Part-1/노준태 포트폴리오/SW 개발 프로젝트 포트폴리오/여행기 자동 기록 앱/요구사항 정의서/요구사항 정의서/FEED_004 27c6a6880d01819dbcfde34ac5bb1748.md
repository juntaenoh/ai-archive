## 2️⃣ 정규화 가중치 계산

---

*1.  아키타입 빈도 가중치 추가* 

```python
# 실제 코드 (feed_svc.py:510-517)

WEIGHT_MIN = 10   # 최소 가중치
WEIGHT_MAX = 100  # 최대 가중치

archetype_weights = {}
if archetype_counts:
    max_count = max(archetype_counts.values())  # 최대 빈도
    for archetype, count in archetype_counts.items():
        # 정규화: WEIGHT_MIN-WEIGHT_MAX 범위로 변환
        weight = int(WEIGHT_MIN + (count / max_count) * (WEIGHT_MAX - WEIGHT_MIN))
        archetype_weights[archetype] = weight

# 예시

# 사용자 로그: 모험가 9개, 교감러 4개, 힐링러 2개
max_count = 9

# 가중치 계산
모험가: 10 + (9/9) * (100-10) = 10 + 1.0 * 90 = 100점
교감러: 10 + (4/9) * (100-10) = 10 + 0.44 * 90 = 50점
힐링러: 10 + (2/9) * (100-10) = 10 + 0.22 * 90 = 30점

# 최종 가중치
final_weights = {
    "모험가": 100,
    "교감러": 50,
    "힐링러": 30
}

***-> 빈도수에 대해서 정규화 진행 (가중치 넘겨주기 위함)***
```

*2. 시간 가중치 추가 : 최신성있는 게시물에 대해서 더 높은 점수를 부여하기 위함* 

```python
# 실제 코드 (feed_svc.py:643-655)

def _calculate_days_ago(self, created_at_str: str) -> int:
    try:
        created_at = datetime.fromisoformat(created_at_str.replace('Z', '+00:00'))
        now = datetime.now(timezone.utc)
        days_ago = (now - created_at).days
        return max(0, days_ago)
    except Exception:
        return 0

# 시간 점수 계산 (feed_svc.py:657-665)
def _calculate_time_score(self, days_ago: int) -> int:
    base_score = self.TIME_SCORE_BASE  # 100
    decay = self.TIME_SCORE_DECAY      # 5
    time_score = max(self.WEIGHT_MIN, base_score - (days_ago * decay))
    return int(time_score)

### 시간 가중치 계산 방식
- **기준 점수**: 100점 (0일 전)
- **감소율**: 하루마다 5점씩 감소
- **최소 점수**: 10점 (더 이상 감소하지 않음)

```

*3. 최종 가중치 → 점수 변환 (여행 스타일 + 최신성 반영된다 생각할 수 있음)*

```python
# 실제 코드 (feed_svc.py:600-625)
for item in filtered_posts:
    archetype = item.get('archetype', '')
    created_at = item.get('created_at', '')

    # 아키타입 가중치
    archetype_weight = weights.get(archetype, self.WEIGHT_DEFAULT)  # 기본값 10

    # 시간 가중치
    days_ago = self._calculate_days_ago(created_at)
    time_score = self._calculate_time_score(days_ago)

    # 최종 점수
    final_score = archetype_weight + time_score

    scored_posts.append({
        'id': item.get('id'),
        'created_at': created_at,
        'archetype': archetype,
        'score': final_score
    })

```

## 4️⃣ 비숙련 사용자 추천 로직: Cold Start 해결

*사용자 분류 → 로그 개수* 

1. **신규 사용자**: 프로필 아키타입에 완전 의존하여 즉시 추천 제공
2. **초보 사용자**: 프로필 우선하되 다른 아키타입도 점진적 노출
3. **성장 사용자**: 행동 패턴과 프로필을 적절히 혼합하여 학습 시작
4. **숙련 사용자**: 완전 행동 기반으로 진정한 개인화 추천

*기준 - 자체 설정* 

- **신규 사용자**: 0개 로그
- **초보 사용자**: 1-3개 로그
- **성장 사용자**: 4-10개 로그
- **숙련 사용자**: 11개 이상 로그

## 1단계: 신규 사용자 (0개 로그)

**> 프로필 기반 추천** 사용 / 사용자가 프로필에서 선택한 아키타입의 게시물이 대부분 추천

**해결 방법** 

**프로필 아키타입**: 100점 (최고 우선순위)

**다른 아키타입들**: 10점 (최소 노출)

---

## 2단계: 초보 사용자 (1-3개 로그)

**> 데이터 부족**: 여전히 행동 패턴 분석에 부족

**> 프로필 의존**: 프로필 아키타입에 의존해야 함

**> 다양성 필요**: 점진적으로 다른 아키타입 노출 필요

**해결 방법 : 프로필 우선 + 다양성 도입**을 사용

**프로필 아키타입**: 80점 (높은 우선순위)

**다른 아키타입들**: 30점 (중간 노출)

---

## 3단계: 성장 사용자 (4-10개 로그)

**> 데이터 축적**: 어느 정도 행동 패턴 분석 가능

**> 혼합 필요**: 프로필과 행동 패턴의 적절한 조합

**해결 방법 : 행동 + 프로필 혼합 추천 사용** 

**행동 패턴**: 70% 비중

**프로필**: 30% 비중

<aside>
💡

**예시**

---

사용자 프로필 "모험가", 7개 로그 (모험가 5개, 교감러 2개)

1. **행동 패턴 가중치 계산**:
    - 모험가: 100점 (가장 많이 작성)
    - 교감러: 40점 (상대적 비율)
2. **프로필 가중치**:
    - 모험가: 60점
3. **혼합 (행동 70% + 프로필 30%)**:
    - 모험가: (100 × 0.7) + (60 × 0.3) = 88점
    - 교감러: (40 × 0.7) + (0 × 0.3) = 28점
</aside>

## 5️⃣ Cold start 문제 정리

---

### 1. 단계별 비교표

---

| 사용자 단계 | 로그 개수 | 전략 | 프로필 가중치 | 다른 아키타입 | 특징 |
| --- | --- | --- | --- | --- | --- |
| **신규** | 0개 | 프로필 기반 | 100점 | 10점 | 프로필 완전 의존 |
| **초보** | 1-3개 | 프로필 우선 | 80점 | 30점 | 점진적 다양성 |
| **성장** | 4-10개 | 혼합 추천 | 60점 (30%) | 행동 기반 (70%) | 학습 시작 |
| **숙련** | 11개+ | 행동 기반 | 무시 | 정규화 가중치 | 완전 개인화 |

### 2. 특징

---

- 점진적 학습
    - **신규 → 초보 → 성장 → 숙련** 단계별 전략
    - 사용자 데이터 축적에 따른 추천 품질 향상

- Cold Start 해결
    - **데이터 부족 시**: 프로필 아키타입 활용
    - **즉시 추천**: 신규 사용자도 빠르게 개인화된 추천 제공
    - **신규 사용자**: 즉시 개인화된 추천 제공
        
        <aside>
        💡
        
        - **데이터 축적**: 추천 품질 점진적 향상
        - **사용자 이탈**: 감소 효과
        </aside>
        

- 자연스러운 전환
    - **단계별 비율 조정**: 프로필 → 행동 패턴으로 자연스러운 전환
    - **사용자 경험**: 갑작스러운 변화 없이 점진적 개선
    
    <aside>
    💡
    
    **단계별 맞춤**: 사용자 수준에 맞는 추천
    
    **자연스러운 학습**: 갑작스러운 변화 없음
    
    **지속적 참여**: 단계별 성장감 제공
    
    </aside>
    

- 다양성 보장
    - **모든 단계**: 다양한 아키타입 노출
    - **최소 보장**: 모든 아키타입에 최소 10점 보장
