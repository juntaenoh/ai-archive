# COMM_001_3

상태: 초안 작성
구분: 커뮤니티
기능 간단 설명: 자동으로 정리된 여정별 사진 앨범 중 대표 이미지를 썸네일로 제공
기능명: 단일 여행기 썸네일 조회
우선 순위: 1순위
상위 항목: COMM_001 (COMM_001%2027c6a6880d0181db96fec4a7a7f76404.md)

| 항목 | 내용 |
| --- | --- |
| **기능 ID** | COMM_001_3 |
| **기능 명** | 사진 앨범 |
| **기능 설명** | 여행 중 업로드되거나 연동된 사진들을 일정 구간별로 자동 분류하여, 여행기 상세 페이지 내에서 앨범 형식으로 제공한다. 썸네일 형태로 전체 여행 사진을 시각적으로 탐색할 수 있으며, 각 사진 클릭 시 전체 보기로 전환된다. 감정 기반 필터링 기능과 연동될 수 있다. |
| **사용자 액션** | 1. 여행기 상세 보기 진입2. “사진 보기” 탭 선택 or 앨범 영역 스크롤3. 썸네일 클릭 시 전체 화면 보기 |
| **시스템 동작** | 1. travel_id 기반으로 연결된 사진 메타데이터 조회
2. 사진별 timestamp 및 장소 정보 기준으로 일정 구간(trip_section)별 분류
~~3. 각 구간별로 썸네일형 사진 목록 구성
4. 감정 태그와 연동되어 관련 필터링도 가능~~ |
| **입력 데이터** | travel_id, photo_url, timestamp, place_name, section_id |
| **출력 데이터** | 사진 썸네일 리스트 (`photo_thumbnail_list`) with section grouping |
| **상태 전이** | 없음 |
| **예외 처리** | 사진이 없는 일정 구간 → “이 장면에 등록된 사진이 없습니다” 안내 표시이미지 로딩 실패 시 → 기본 이미지 또는 'X' 아이콘 대체 |
| **비고** | 썸네일 선택 시 full-screen carousel(슬라이드) 제공 가능사진에 감정 태그 연동되면 필터 기능 추가 구현 고려 |

*사진첩 동의를 받아서, 사진을 촬영할 때 마다 근처 관광지에 대한 사진들로 기록하고 접근 할 수 있도록 하면 되는 건가?* 

### 📌 UX 흐름 예시

```
[여행기 상세 보기 진입]
→ 각 정리 카드 아래 앨범 섹션 노출
→ “속초해변” 일정 구간: 사진 10장, 
→ 사진을 수동으로 입력하여 해당 관광지에 대한 사용자 카드 업로드 
```
