# RECD_001

상태: 초안 작성
구분: 여행기 기록
기능 간단 설명: "여행 기록 시작" 버튼을 눌러 새로운 여행기 세션을 생성
기능명: 여행 기록 시작
우선 순위: 1순위

| 항목 | 내용 |
| --- | --- |
| **기능 ID** | RECD_001 |
| **기능 명** | 여행 기록 시작 |
| **기능 설명** | 사용자가 여행을 시작하기 위해 **`여행 기록 시작`** 버튼을 누르면 새로운 여행 세션 하나가 생성되고, 
이후 사용자의 여행 데이터가 해당 세션에 기록된다. |
| **사용자 액션** | **`여행 기록 시작`** 버튼 클릭 |
| **시스템 동작** | 1. 사용자의 **`현재 위치`**와 **`현재 시간`** 수집
2. 새로운 여행기 ID 생성
3. 여행기 상태를 기록 중인 상태로 초기화
4. 여행기 시작 시각, 현재 위치, 디바이스 정보 기록
5. 생성된 여행기 ID를 로컬/세션에 저장하여 후속 기록과 연동
6. 타임라인 화면 혹은 감성 입력 화면으로 이동 |
| **입력 데이터** | 없음 (클릭 이벤트만) |
| **출력 데이터** | 여행기 ID, 기록 중인 상태 표시(기록 중, 기록 중지 등), 타임 스탬프, GPS 좌표 등 |
| **상태 전이** | `none → recording` (기록 시작) |
| **예외 처리** | GPS 권한 오류 : 위치 권한 요청 및 경고 메시지 
GPS 신호 불량 : 
네트워크  연결 불안정 : 로컬에 임시 저장 후 백엔드 sync 대기 상태 유지
중복 여행 시작 요청 : "기록 중인 여행이 존재합니다"  |
| **비고** | 최초 여행 시작 시점은 모든 데이터 기록의 기준점이므로, 정확한 시간과 위치 저장이 핵심. 
추후 “기록 중 중단” 기능과 연결될 수 있음. |

## ☑️ 개발자/기획자 참고사항

---

| 역할 | 주의 사항 |
| --- | --- |
| **백엔드** | 여행기 시작 시점마다 새로운 여행기 ID를 생성해야 하며, 
이후 모든 기록(감정 문장, 여행 사진 등)이 해당 여행기 ID와 연결되어야 함. |
| **프론트엔드** | 위치 권한 체크 필요. 버튼 클릭 시 사용자의 위치를 즉시 수집해야 하며, 실패 시 명확한 피드백 필요. 
UI 전환 효과도 고려 (예: 애니메이션 효과로 기록 시작 느낌 부여) |
| **AI 처리** | 이후 감정 태깅, 사진/날씨/지도 자동 연동 시 이 travel_id 기준으로 묶어서 처리해야 함. |

## ☑️ 시퀀스 다이어그램 (요약 흐름)

---

```
[User] → 클릭 → [Frontend] → 위치 요청 → [Backend]
→ travel_id 생성 → DB 저장 → 상태값: recording
→ travel_id 리턴 → 화면 전환
```
