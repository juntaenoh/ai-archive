# RECD_004

상태: 초안 작성
구분: 여행기 기록
기능 간단 설명: 여행 기록 중, 사용자가 촬영한 사진을 기록
기능명: 여행 사진 기록
우선 순위: 1순위

| 항목 | 내용 |
| --- | --- |
| **기능 ID** | RECD_004 |
| **기능 명** | 여행 사진 기록 |
| **기능 설명** | 사용자가 여행 중 촬영한 사진을 자동 또는 수동으로 연동하여, 여행기 타임라인에 포함시킨다. 
촬영 시점의 위치·시간 정보와 함께 감정 입력, 날씨, 장소 등과 연결되어 여행의 감성 기록을 더욱 풍부하게 만들어준다. |
| **사용자 액션** | 1. 사용자가 직접 사진을 선택하여 업로드 (수동)
2. 기기 내 갤러리에서 메타데이터를 기준으로 자동 추천 (자동) |
| **시스템 동작** | 1. 사진 선택 또는 감정 입력 시점 기준으로 디바이스에서 촬영된 사진 리스트를 조회
2. 사진의 메타데이터(촬영 시각, 위치 좌표)를 기준으로 해당 여행기 세션(`travel_id`) 내에 자동 연동
3. 위치 좌표는 기존 `recd_002`의 관광지 추정 위치와 매칭되어 `place_id` 및 `place_name`으로 추상화 처리
4. 사진은 S3 등 외부 스토리지에 저장되고, URL을 travel_log와 연결
5. UI에서는 타임라인 구성 시 텍스트 기록과 함께 해당 시점의 사진을 카드 형태로 시각화 |
| **입력 데이터** | 사진 파일, 사진의 메타데이터(EXIF: timestamp, GPS), travel_id |
| **출력 데이터** | `photo_log_entry` 객체 → `photo_id`, `travel_id`, `place_id`, `place_name`, `photo_url`, `timestamp`, `matched_distance`, `entry_type: "사진"` |
| **상태 전이** | 없음 |
| **예외 처리** | EXIF 메타데이터 없음: 사용자가 직접 장소·시간 선택하거나 최근 감정 입력과 연결관광지 매칭 실패: “이름 없는 장소”로 연결 또는 수동 선택 
유도저장 실패: 재시도 또는 오프라인 상태 저장 후 sync 대기 |
| **비고** | 사용자 경험을 위해 최근 1시간 내 사진을 자동 추천하는 기능 필요사진과 감정 문장이 함께 기록될 경우, AI가 감정 중심 카드 콘텐츠로 묶어 처리 가능위치정보법 준수를 위해 실제 좌표 저장은 금지, `recd_002`와 동일한 위치 근사 처리 방식을 따름 |

## ☑️ 개발자/기획자 참고사항

---

| 역할 | 주의 사항 |
| --- | --- |
| **백엔드** | 사진은 S3 등 외부 저장소에 저장하고, URL만 travel_log에 연동. 단일 API로 이미지 업로드와 로그 연결 처리. |
| **프론트엔드** | 사진 선택 시점의 메타데이터에서 시간·위치 정보를 읽어 관련 감정 로그와 자동 연동. UI에서 최근 사진 자동 추천 기능 제공. |
| **AI 처리** | 사진 내용 분석 기반 감정 태깅은 선택 기능으로 확장 가능. 사용자 감정 입력과 시각정보를 연결하여 카드 콘텐츠 생성 시 활용. |
| **디자이너** | 감정 + 사진이 타임라인에 카드 형태로 함께 구성되도록 UI/UX 설계. 앨범 뷰와 타임라인 뷰는 다른 구성 필요. |

### 📌 예시 동작 시나리오

```
[사용자] → 사진 업로드 → [앱] → 메타데이터 추출
→ 촬영 시각과 위치 기준으로 관광지 근사 매칭
→ 장소명·시간과 함께 타임라인 카드 생성
→ 사진은 S3에 저장, URL로 travel_log에 연동
```
