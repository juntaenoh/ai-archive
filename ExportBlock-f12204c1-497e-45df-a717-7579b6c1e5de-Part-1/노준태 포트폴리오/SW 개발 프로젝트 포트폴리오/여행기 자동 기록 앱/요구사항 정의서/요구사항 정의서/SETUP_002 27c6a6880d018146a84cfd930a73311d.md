# SETUP_002

상태: 초안 작성
구분: 개인 설정
기능 간단 설명: 본인이 생성한 단일 여행기에 대한 데이터 편집
기능명: 나의 종료된 단일 여행기 편집
우선 순위: 2순위

| 항목 | 내용 |
| --- | --- |
| **기능 ID** | SETUP_002 |
| **기능 명** | 나의 종료된 단일 여행기 편집 |
| **기능 설명** | 자동 기록 외에도, 사용자가 원하는 시점에 감정 문장, 사진, 장소 정보 등을 직접 입력하여 여행기 타임라인에 수동으로 추가할 수 있는 기능이다. 특정 구간에 콘텐츠를 보완하거나, 자동 분할되지 않은 순간을 수동으로 기록하고자 할 때 사용된다. |
| **사용자 액션** | 타임라인 하단 또는 특정 구간에서 “내용 추가” 버튼 클릭→ 감정 문장/사진/장소 선택 |
| **시스템 동작** | 1. 사용자가 수동 입력 모드 진입
2. 입력 항목 선택: 감정 문장 / 사진 / 장소
3. 입력 시각 자동 기록 또는 사용자가 선택 가능
4. 해당 travel_id와 연결된 타임라인에 새로운 카드 생성 또는 기존 구간에 병합
5. 수동 입력은 별도로 표시 (아이콘 또는 라벨) |
| **입력 데이터** | 감정 문장, 사진 파일, 장소명 (선택), 시각 (기본 현재 시각) |
| **출력 데이터** | `manual_log_entry` 객체 → travel_id, content_type, content_data, timestamp, source: "manual" |
| **상태 전이** | 없음 (기록 완료 후 수정/삭제는 허용) |
| **예외 처리** | 사진 없이 감정만 입력: 이미지 없이 카드 구성감정 없이 사진만 입력: 사진 중심 카드 구성입력 취소 시 → 기록 없음 |
| **비고** | 수동 추가 항목은 ‘✍ 수동 기록’ 또는 ‘🖐 사용자 추가’와 같은 UI 표시로 구분정리 카드 구성 시 자동 기록과 함께 정렬되어 표시됨 (시간순)여행 종료 후 회고 단계에서도 수동 추가 가능 |

### 📌 UX 흐름 예시

```
**[여행기 타임라인 보기 화면]**

→ 하단 “+ 수동 기록 추가” 버튼 클릭
→ 감정 입력: “이 순간 너무 행복했다”
→ 사진 추가: 웃고 있는 셀카
→ “속초중앙시장” 장소 선택
→ 저장 → 타임라인에 새로운 카드 추가됨
```

### 개발자/기획자 참고사항

| 역할 | 주의 사항 |
| --- | --- |
| **백엔드** | `manual_log` 테이블 또는 `travel_log`에 source: "manual"로 구분하여 저장. 시간순 정렬 시 자동 기록과 병합되도록 처리. |
| **프론트엔드** | 수동 추가 UI는 입력 폼, 미리보기 카드, 업로드 버튼으로 구성. 입력 후 즉시 타임라인에 반영되도록 비동기 처리. |
| **디자이너** | 수동 추가 카드는 “✍ 직접 남긴 기록”이라는 인상을 줄 수 있는 라벨 또는 테두리 스타일 사용 권장 |
| **AI 처리** | 사용자가 작성한 감정 문장도 다른 기록과 동일하게 감정 태그 추출 및 감정 요약에 포함 가능 |
