# RECD_006

상태: 초안 작성
구분: 여행기 기록
기능 간단 설명: 여행 기록 중, 일정 거리 이동 또는 일정 시간 체류 시, 여정 구간이 자동 분리되어 기록
기능명: 여행 구간 분할
우선 순위: 1순위

| 항목 | 내용 |
| --- | --- |
| **기능 ID** | RECD_006 |
| **기능 명** | 여행 구간 분할 |
| **기능 설명** | 여행 중 수집된 위치/시간 데이터를 바탕으로 여정을 자동으로 구간별로 분할하여 여행 타임라인을 생성한다.
일정 거리 이상 이동했거나, 일정 장소에 일정 시간 이상 체류했을 경우, 또는 감정/사진 등의 주요 활동이 발생한 경우 새로운 일정 구간으로 판단한다. 이를 통해 사용자 개입 없이도 “여정의 장면” 단위로 자동 기록이 분리된다. |
| **사용자 액션** | 없음 (자동 처리), 또는 사용자가 수동 분할 기준 설정 가능 (옵션) |
| **시스템 동작** | 1. `travel_id`에 해당하는 위치 기록, 감정 기록, 사진 기록, 시간 간격 등을 수집
2. 다음 조건 중 하나 이상 충족 시 새로운 일정 구간(split) 생성: a. 500m 이상 이동 b. 30분 이상 정체(체류) c. 감정 기록 발생
d. 사진 업로드 발생
3. 각 일정 구간은 시작 시각, 종료 시각, 대표 장소(place_name), 대표 감정 요약값으로 구성
4. 자동 분할 구간은 타임라인 또는 카드 형태로 시각화 |
| **입력 데이터** | 위치 로그(GPS→place_name), 감정 기록, 사진 기록, 타임스탬프, travel_id |
| **출력 데이터** | `trip_section_entry` 객체 → `section_id`, `travel_id`, `start_time`, `end_time`, `place_name`, `summary_tags`, `media_count` 등 |
| **상태 전이** | 없음 |
| **예외 처리** | 모든 기록이 한 장소에 집중된 경우 → "단일 일정"으로 판단위치 정보 없음 → 시간 기반 분할 (예: 2시간 이상 간격)감정/사진 없음 → 최소 거리 기준 분할만 적용 |
| **비고** | 분할 기준은 기본값으로 제공하되, 사용자 설정에서 커스터마이징 가능 (예: "나는 1시간 단위로 보고 싶어요")AI가 일정별 감정 요약, 사진 대표 선정 등을 자동 수행하여 카드 콘텐츠로 구성 가능 |

## ☑️ 개발자/기획자 참고사항

---

| 역할 | 주의 사항 |
| --- | --- |
| **백엔드** | 위치·시간·감정·사진 로그를 통합하여 분석 후, 일정 구간을 travel_id 단위로 생성.
조건 기반 rule-engine 또는 간단한 ML 기반 segmentation도 고려 가능. |
| **프론트엔드** | 일정 구간이 변경되면 타임라인에 카드 단위로 반영. 사용자가 구간 병합/분할 요청 시 편집 기능도 고려 |
| **디자이너** | 구간별 대표 장소명/감정/이미지가 보이는 카드 UI 구성. "여정의 장면들" 느낌을 주는 콘텐츠 형태가 중요 |
| **AI 처리** | 일정 구간별 감정 요약 또는 사진 대표 컷 자동 선정 기능을 추가할 수 있음 (예: 감정 태그 중 가장 자주 등장한 키워드 표시) |

### 📌 예시 동작 시나리오

```
[사용자 이동] → [위치 기록 ①] 속초해변 (10:00)
→ [30분 이상 체류] → 감정 기록: "시원하다"
→ [위치 기록 ②] 봉포항 (11:30)
→ 사진 업로드: 바닷가 풍경

→ 시스템은 이를 다음과 같이 자동 분할
1️⃣ 10:00~11:00: 속초해변 (감정: 시원하다, 체류)
2️⃣ 11:30~12:00: 봉포항 (사진 업로드)
```
