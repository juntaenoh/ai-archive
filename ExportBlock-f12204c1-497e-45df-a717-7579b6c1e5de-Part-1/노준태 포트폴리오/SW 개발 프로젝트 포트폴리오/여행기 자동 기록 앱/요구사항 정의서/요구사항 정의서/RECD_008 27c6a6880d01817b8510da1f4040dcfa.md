# RECD_008

상태: 초안 작성
구분: 여행기 기록
기능 간단 설명: 여행 기록 중, 기록 중인 내용이 중간에 자동으로 저장되어 앱 비정상적으로 종료된 이후에도 기록중인 내용 복원 가능
기능명: 여행 기록 임시 저장
우선 순위: 2순위

| 항목 | 내용 |
| --- | --- |
| **기능 ID** | RECD_008 |
| **기능 명** | 여행 기록 임시 저장 |
| **기능 설명** | 여행기록이 진행 중일 때 입력된 감정, 사진, 위치 등의 데이터를 사용자가 명시적으로 저장하지 않아도 자동으로 주기적으로 저장한다. 앱이 강제 종료되거나 네트워크가 불안정한 상황에서도, 다시 실행 시 동일한 `travel_id` 기준으로 기록을 복원할 수 있다. |
| **사용자 액션** | 사용자는 명시적으로 저장하지 않음 → 자동 저장 발생 |
| **시스템 동작** | 1. 기록 상태가 `recording`일 때 일정 주기(예: 30초 ~ 1분)마다 로컬/세션/서버에 자동 저장
2. 감정 기록, 사진 업로드, 위치 기록 등 주요 이벤트 발생 시에도 트리거 기반 저장
3. 앱 종료 또는 중단 시 자동 저장된 데이터 유지
4. 앱 재실행 시 `recording` 상태의 여행기 존재 여부 확인 후 복원 제안 |
| **입력 데이터** | 감정 문장, 감정 태그, 사진 메타데이터, place_name, timestamp 등 |
| **출력 데이터** | `autosave_snapshot` → travel_id, 기록 중인 로그들, 마지막 저장 시각 |
| **상태 전이** | 없음 (recording 상태에서만 작동) |
| **예외 처리** | 디바이스 저장소 부족 → “저장 공간 부족” 알림오프라인 상태 → 로컬 저장 후 서버 sync 대기복원 실패 시 → 사용자에게 “부분 복원됨” 안내 |
| **비고** | 자동 저장 주기와 저장 항목은 설정 가능서버와의 동기화 주기 또는 로컬 캐시 전략 필요여러 디바이스 간 동기화는 지원하지 않음 (단말 기준) |

## ☑️개발자/기획자 참고사항

---

| 역할 | 주의 사항 |
| --- | --- |
| **백엔드** | `autosave_snapshot` API를 별도로 구성하여 travel_id 단위 저장. 서버와 로컬 상태 불일치 시 우선순위 결정 필요 (보통 로컬 우선). |
| **프론트엔드** | 백그라운드 진입, 감정 입력 후 10초 내, 사진 업로드 등 이벤트 기반으로 저장 요청 발생. 앱 재실행 시 복원 팝업 UI 제공 |
| **디자이너** | 복원 팝업 시 UX 신뢰성을 줄 수 있는 문구 구성: “이전 기록이 자동으로 저장되어 있어요. 이어서 계속할까요?” |
| **AI 처리** | 감정 태깅, 사진 분석 결과도 snapshot에 포함 필요. 재시작 시 동일한 콘텐츠 흐름 보장해야 함 |

### 📌 예시 동작 흐름

```
[사용자] 감정 입력: "시원하다"
→ 시스템: autosave snapshot 생성 (travel_id, 감정 로그, 장소명, 시간 저장)

[사용자] 사진 업로드
→ 즉시 autosave → 서버 or 로컬에 저장

[앱 강제 종료]
→ 다음 실행 시: “기록 중이던 여행이 있습니다. 이어서 계속하시겠어요?” → 확인 시 복원

```
