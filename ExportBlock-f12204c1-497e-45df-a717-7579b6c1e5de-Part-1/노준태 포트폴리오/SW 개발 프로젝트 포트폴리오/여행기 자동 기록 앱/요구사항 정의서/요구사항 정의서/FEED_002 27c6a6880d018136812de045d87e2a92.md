# FEED_002

상태: 초안 작성
구분: 커뮤니티
기능 간단 설명: 사용자 취향 및 감성 키워드, 아키타입 등을 기반으로 다른 사용자의 여행기들 중, 추천하는 여행기 목록을 조회
기능명: 여행기 추천 기능
우선 순위: 2순위

*rec_001 → 비기능적 명세됨 + 추천 방식 추후 이야기*  

| 항목 | 내용 |
| --- | --- |
| **기능 ID** | FEED_002 |
| **기능 명** | 여행기 추천 기능 |
| **기능 설명** | 사용자 개인의 감정, 관심 태그, 장소 선호도, 아키타입 성향 등을 기반으로 콘텐츠 피드를 자동 정렬한다. 사용자의 기록 기반 벡터와 콘텐츠별 벡터 간 유사도를 계산하여, 우선순위를 부여하고 가장 공감도 높은 콘텐츠를 상단에 노출한다. |
| **사용자 액션** | 앱 접속, 피드 진입 시 자동 노출 |
| **시스템 동작** | 1. 사용자 감성·장소·아키타입 벡터 불러오기
2. 전체 피드 카드 메타데이터 불러오기 (태그, 장소, 감정 문장 등)
3. 각 카드와 사용자 간 유사도 계산
4. 유사도 점수 기반으로 우선순위 정렬
5. 피드 카드 리스트 반환 |
| **입력 데이터** | `user_emotion_vectoruser_place_vectoruser_archetype_scorecard_metadata_list` |
| **출력 데이터** | 정렬된 `feed_card_list` (score 포함) |
| **예외 처리** | 신규 사용자 → 인기 태그 기반 피드 노출유사 콘텐츠 부족 → 랜덤성 or 인기도 주기  |
| **비고** | 사용자의 상호작용(좋아요, 댓글, 스크롤 깊이 등)을 반영해 추천 벡터를 지속적으로 업데이트함. 다양한 아키타입 혼합 사용자도 유연하게 대응 가능. |

### 유사도 계산 알고리즘 예시

```python

score = (
  emotion_similarity * 0.5 +
  tag_overlap_score * 0.2 +
  place_similarity * 0.2 +
  archetype_alignment * 0.1
)

- 감정/태그/장소는 벡터로 변환해 cosine similarity 계산
- 아키타입 정렬 우선 콘텐츠는 +α 점수 부여
```

### 예시

| 사용자 벡터 | 콘텐츠 카드 벡터 | 유사도 |
| --- | --- | --- |
| `#고요함`, `#해변`, `#혼자여행` | `#노을`, `#고요함`, `#혼자여행` | 0.92 (최상단 노출) |
| `#사진여행`, `#핫플투어` | `#사진중심`, `#시장구경`, `#가족여행` | 0.48 (하위 노출) |

### 개인화 방식

| 방식 | 설명 |
| --- | --- |
| 감정 유사도 | 감성 문장, 태그 기반 유사도 계산 |
| 장소 선호도 | 방문한 장소와 비슷한 위치의 콘텐츠 우선 |
| 아키타입 매칭 | 동일 유형의 사용자가 좋아한 콘텐츠 우선 |
| 반응 기반 학습 | 좋아요/댓글/스크롤 기록으로 추천 벡터 지속 업데이트 |
