# RECD_012

상태: 초안 작성
구분: 여행기 기록
기능 간단 설명: 여행 기록 중, 기록 중인 현 상태 자체를 화면에 표시하고, “기록 중단”, “기록 완료” 상태로 변경 가능
기능명: 여행 기록 상태 조회 및 변경
우선 순위: 1순위

| 항목 | 내용 |
| --- | --- |
| **기능 ID** | RECD_012 |
| **기능 명** | 여행 기록 상태 조회 및 변경 |
| **기능 설명** | 사용자가 여행기록을 시작한 이후, 현재 기록이 **진행 중인지, 일시 중단됐는지, 종료됐는지**를 앱 내 명확히 표시한다. 
이를 통해 사용자 스스로 현재 상태를 인식하고 적절한 조작(재시작, 종료 등)을 할 수 있도록 유도한다. 상태는 UI에 텍스트, 색상, 아이콘 등으로 명시되며 시스템 내부적으로는 상태 값(`recording`, `paused`, `completed`)으로 관리된다. |
| **사용자 액션** | - “기록 시작” 버튼 클릭- “기록 중지” 또는 “종료” 버튼 클릭- 앱을 백그라운드/종료 후 재실행 |
| **시스템 동작** | 1. 사용자의 여행기록 상태(`record_status`)를 세 가지 중 하나로 유지
2. 앱 내 상단 위젯 또는 홈 화면에 상태 UI 표시: 
① 진행 중: `recording` ② 일시 중지: `paused` ③ 기록 종료: `completed`
3. 상태 변화 시 사용자에게 적절한 알림 또는 안내 제공
4. 앱 종료 후 재진입 시, 기록 중인 여행이 있을 경우 `recording` 상태를 복구 표시 |
| **입력 데이터** | 사용자 조작(버튼 클릭, 앱 재실행 등), 시스템 상태 |
| **출력 데이터** | 기록 상태 값 (`recording`, `paused`, `completed`), 상태 표시 UI 요소 (텍스트, 색상, 아이콘) |
| **상태 전이** | `none → recording → paused ↔ recording → completed` |
| **예외 처리** | 기록 중이지만 위치 권한 해제 → 상태는 `recording`, 알림 표시앱 강제 종료 후 재실행 → `recording` 상태 복구 및 재표시중복 시작 시도 → “이미 기록 중인 여행이 있어요” 경고 |
| **비고** | 상태 값은 클라이언트와 서버 모두 관리기록 상태에 따라 UI의 기능 제한이 있을 수 있음 (예: 기록 종료 후에는 감정 입력 불가)백그라운드 상태로 오래 있으면 자동 일시 중지 처리를 고려 가능 |

## ☑️ 상태 정의 및 전이 흐름

---

| 상태 | 설명 |
| --- | --- |
| `none` | 여행기록 시작 전 (앱 초기 상태) |
| `recording` | 여행기록이 진행 중인 상태 |
| `paused` | 앱이 종료되었거나 백그라운드로 일정 시간 이상 유지되어 중지된 상태 |
| `completed` | 사용자가 “기록 종료”를 명시적으로 선택한 상태 |
