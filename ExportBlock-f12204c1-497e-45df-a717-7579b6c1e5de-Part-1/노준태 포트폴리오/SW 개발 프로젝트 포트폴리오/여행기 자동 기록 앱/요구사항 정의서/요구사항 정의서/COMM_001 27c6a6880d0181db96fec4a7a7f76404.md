# COMM_001

상태: 초안 작성
구분: 커뮤니티
기능 간단 설명: 종료된 단일 여행기에 대한 타임라인, 여행 경로, 여행 사진, 여행 메모 등 상세 내용 조회
기능명: 단일 여행기 상세 조회
우선 순위: 1순위
하위 항목: COMM_001_1 (COMM_001_1%2027c6a6880d018151844bf62f30a500e1.md), COMM_001_2 (COMM_001_2%2027c6a6880d0181a882a0db7e1a76dcdd.md), COMM_001_3 (COMM_001_3%2027c6a6880d0181108089d3633134726a.md)

| 항목 | 내용 |
| --- | --- |
| **기능 ID** | COMM_001 |
| **기능 명** | 단일 여행기 상세 조회 |
| **기능 설명** | 사용자의 여행 기록이 종료되면, travel_id 단위로 수집된 일정 구간들을 시간 순서에 따라 자동으로 구성하여 감성 타임라인 형식으로 보여준다. 각 일정 구간은 대표 장소, 감정 문장 또는 태그, 대표 사진 등을 포함한 카드 형태로 표시되며, 사용자는 이를 통해 여행의 전체 흐름을 감성적으로 되짚어볼 수 있다. |
| **사용자 액션** | 마이페이지 또는 홈에서 ‘여행기 보기’ 클릭→ 특정 travel_id 선택 |
| **시스템 동작** | 1. travel_id 기반으로 기록된 `trip_section` 구간들을 시간순으로 정렬
2. 각 구간에 포함된 감정, 장소, 사진 등을 조합하여 카드 UI 자동 생성
3. 타임라인 뷰에 카드 단위로 나열
4. 각 카드 클릭 시 해당 구간의 상세 뷰 진입
5. 날짜별 구분 또는 요약 문구(“속초에서의 하루”) 추가 구성 가능 |
| **입력 데이터** | travel_id, trip_section, place_name, photo_url, emotion_tag, timestamp |
| **출력 데이터** | 카드 형식 콘텐츠 (`feed_card_entry`) → place_name, 감정 요약, 대표 이미지, 시간 범위 등 |
| **상태 전이** | 없음 (완료된 기록을 조회만 함) |
| **예외 처리** | 일정 구간 없음 → “기록된 일정이 없습니다” 안내감정/사진 없는 구간 → “내용 없음” 카드 or 최소 정보만 노출이미지 로딩 실패 → 기본 일러스트로 대체 |
| **비고** | 카드 구성 기준: 대표 장소명, 감정 태그, 대표 이미지, 시간카드의 순서는 시간 순 + 감정의 다양성 고려하여 자동 정렬카드 구성 후 사용자 수동 편집 기능은 향후 확장 가능 |

| 요소 | 설명 | 예시 |

| 대표 이미지 | 해당 구간 내 대표 이미지 선정 | 사진 or 랜덤 일러스트 |

| ~~이동 정보 (선택)~~ | 전후 장소 대비 거리 or 시간 | “이동 거리 500m, 다음 장소: 중앙시장” |

### 🧠 개발자/기획자 참고사항

| 역할 | 주의 사항 |
| --- | --- |
| **백엔드** | 일정 구간(trip_section)과 관련된 감정/사진 로그를 시간순으로 조회 후, 카드 단위 JSON 반환 |
| **프론트엔드** | feed_card 구성 요소를 기반으로 UI 타임라인 렌더링. 날짜 구분 헤더 및 카드 클릭 시 상세 보기 연동 |
| **디자이너** | 카드 비주얼은 장소 중심 + 감정 키워드 강조 + 이미지 배경 레이아웃. 일관된 UX 흐름 유지 필요 |
| **AI 처리 (선택)** | 감정 문장을 요약하거나, 대표 장소 또는 이미지 자동 선정 가능 (가장 감정이 뚜렷한 지점 기준) |
