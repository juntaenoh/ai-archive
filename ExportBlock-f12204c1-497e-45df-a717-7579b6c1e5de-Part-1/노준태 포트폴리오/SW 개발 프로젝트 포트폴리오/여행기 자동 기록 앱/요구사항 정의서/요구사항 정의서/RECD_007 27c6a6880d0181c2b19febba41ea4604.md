# RECD_007

상태: 초안 작성
구분: 여행기 기록
기능 간단 설명: 여행 기록 중, 기록된 여정을 타임라인(시간 순서) 또는 카카오 API 를 활용해 지도에 경로 표시
기능명: 여행 타임라인 기록
우선 순위: 2순위

| 항목 | 내용 |
| --- | --- |
| **기능 ID** | RECD_007 |
| **기능 명** | 타임라인 / 지도 시각화 |
| **기능 설명** | 사용자의 여행 기록(`travel_id`)을 시간 순 또는 지도 위에 시각적으로 표현한다. 
감정 기록, 장소 정보, 사진, 일정 구간 등을 기반으로 카드 형태의 타임라인과 지도 경로 UI를 구성하며, 사용자 경험을 감성적으로 풍부하게 구성하는 핵심 인터페이스 역할을 한다. |
| **사용자 액션** | 타임라인 보기 진입, 지도 보기 선택, 날짜 필터 선택, 타임라인 카드 클릭 |
| **시스템 동작** | 1. 여행기(travel_id) 내의 일정 구간(`trip_section`)과 관련된 모든 로그(감정, 장소, 사진)를 시간 순으로 정렬
2. 타임라인 형식: 구간별 대표 장소, 감정 태그, 대표 이미지 등을 포함한 카드 UI 구성
3. 지도 형식: 구간별 장소 좌표를 기반으로 경로를 지도 위에 폴리라인으로 연결
4. 날짜 또는 구간 필터 적용 시 해당 범위만 시각화
5. 타임라인 카드 클릭 시 해당 구간 상세 페이지 진입 (감정 문장, 사진 모음 등 포함) |
| **입력 데이터** | `travel_id`, `trip_section`, `place_name`, `timestamp`, `photo_url`, `emotion_tags`, `place_coord` (좌표는 저장하지 않고 사용만 함) |
| **출력 데이터** | 시각적 카드 UI (타임라인용), 지도상 경로 (Polyline), 장소 마커, 대표 감정 요약 |
| **상태 전이** | 없음 |
| **예외 처리** | 구간 데이터 없음: “기록된 일정이 없어요” 
안내사진/감정 없는 구간: “이 구간에는 내용이 없어요. 직접 추가해보세요” 
카드 노출지도 상 장소 좌표 오류: 지도 표기 제외, 타임라인 상단 경고 |
| **비고** | 감정 태그 색상 시각화, 거리/시간 필터 등 추가 기능 고려지도 상 이동 경로는 장소 간 연결선으로 간결하게 표현 (정확한 궤적 저장은 하지 않음)지도 기능은 카카오맵, Leaflet 등 라이브러리 활용 예정 |

## ☑️개발자/기획자 참고사항

---

| 역할 | 주의 사항 |
| --- | --- |
| **백엔드** | 일정 구간별 장소 정보, 감정 요약, 대표 이미지 URL, 시간 범위를 정리해서 API로 전달 |
| **프론트엔드** | 지도 뷰와 타임라인 뷰 전환 기능, 날짜 필터, 카드 클릭 시 상세 진입 등 인터랙션 UX 설계 필요 |
| **디자이너** | 카드 구성은 대표 장소명, 감정 키워드, 사진이 1:1:1 비율로 고르게 담기도록 디자인. 지도 뷰는 경로 시각화 + 마커 디자인 고려 |
| **AI 처리** | 일정 구간별 감정 요약 문장 또는 이모티콘 자동 생성 가능. 향후 “이 장면은 이런 기분이었어요” 제안 기능으로 확장 가능 |

### 📌 예시 동작 시나리오

```
[사용자] → 여행기 보기 진입

1. [타임라인 뷰]: 속초해변(감정: 시원함, 사진 있음) → 봉포항(감정: 설렘, 사진 있음) → 중앙시장(감정: 혼잡, 사진 없음)
2. [지도 뷰]: 위 장소들을 순서대로 연결한 지도 경로 표출 + 장소별 감정 마커 표시
3. [카드 클릭]: 해당 구간 상세 기록 페이지 진입
```

### ✅ UX 시나리오 예시

| 화면 구성 | 기능 |
| --- | --- |
| 타임라인 뷰 | “장면 단위 카드” (장소명 + 감정 태그 + 대표 이미지 + 시간) |
| 지도 뷰 | “이동 경로 선 + 장소별 마커 + 날짜 필터” |
| 날짜 선택기 | 특정 날짜만 선택 시 해당 구간만 표시 |
| 구간 상세 진입 | 감정 문장 전체, 전체 사진 모음, 장소 설명 등 포함 |

### ✅ 확장 가능 요소

| 확장 항목 | 설명 |
| --- | --- |
