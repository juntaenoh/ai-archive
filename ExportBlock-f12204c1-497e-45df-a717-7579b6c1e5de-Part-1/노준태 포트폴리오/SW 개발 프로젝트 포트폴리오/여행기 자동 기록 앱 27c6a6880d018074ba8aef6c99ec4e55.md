## 📌 프로젝트 개요

여행 중 사진은 많이 찍지만, 나중에 스토리로 정리되지 않는 문제를 해결하기 위해 기획한 토이 프로젝트. GPS/위치 기반으로 여행 기록을 자동 태깅하여 사용자가 별도의 입력 없이도 여행기가 자동으로 완성되는 앱.

## 👥 팀 구성

- 4인 팀 프로젝트
- 담당: 기획 + 프론트엔드 (Flutter)

## 🔍 문제 정의

- 여행 중 사진은 많이 남기지만, 여행 후 기억이 흐려지면서 기록의 품질이 낮아지는 문제
- 기록 자체가 귀찮아서 아예 남기지 않게 되는 사용자 행동 패턴 발견

## ⚙️ 핵심 기능 및 구현

- GPS/위치 기반 자동 태깅 — 사용자 위치를 감지해 방문 장소를 자동으로 기록
- 사용자 입력 최소화 UX 설계 — 기록 부담을 줄이는 것이 핵심 설계 원칙
- Flutter 기반 앱 전반 기능 개발 담당

## 💡 배운 점 / 회고

- 기술 구현보다 문제 정의와 올바른 해결책 선택이 더 어렵다는 것을 깊이 체감
- 현재 시점에서 재구현한다면 LLM API를 활용해 사진/위치 데이터를 자동으로 스토리텔링하는 방향으로 발전시킬 것
